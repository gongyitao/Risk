# 策略分析报告功能说明

## 📊 功能概述

策略E+智能化系统的报告生成模块已经集成了丰富的初始化数据，可以根据不同的业务场景生成专业的策略分析报告。

---

## 🎯 支持的报告类型

### 1️⃣ 年轻用户风险拦截策略回测报告

**场景描述**：针对18-25岁年轻用户，收入较低且入网时间短的高风险群体进行拦截

**策略规则**：
```
IF (年龄 < 25岁) AND (收入 < 5000元) AND (入网时长 < 3个月) THEN 拒绝
```

**预期效果**：
- ✅ 坏账率降低：25%
- ✅ KS值提升：18.75%
- ⚠️ 通过率下降：5.6%
- ✅ 综合评价：显著

**数据规模**：45,678笔申请

**如何触发**：
```
在报告生成模块中输入或点击：
"生成年轻用户风险拦截策略回测报告"
```

---

### 2️⃣ 多头借贷客户拦截策略回测报告

**场景描述**：针对频繁申请借款、同时在多个机构借款的高风险客户进行拦截

**策略规则**：
```
IF (近3个月申请次数 >= 5次) OR (当前在贷机构数 >= 3个) THEN 拒绝
```

**预期效果**：
- ✅ 坏账率降低：42.1%（优秀）
- ✅ KS值提升：44.8%
- ⚠️ 通过率下降：17.3%
- ✅ 综合评价：优秀

**数据规模**：52,341笔申请

**如何触发**：
```
在报告生成模块中输入包含"多头"或"借贷"的问题：
"生成多头借贷客户拦截策略回测报告"
```

---

### 3️⃣ 低收入客群准入策略优化报告

**场景描述**：针对月收入较低但申请高额借款的客户，限制其借款金额

**策略规则**：
```
IF (月收入 < 4000元) AND (借款金额 > 月收入×3) THEN 拒绝
```

**预期效果**：
- ✅ 坏账率降低：29.2%
- ✅ KS值提升：14.7%
- ⚠️ 通过率下降：9.0%
- ✅ 综合评价：显著

**数据规模**：38,924笔申请

**如何触发**：
```
在报告生成模块中输入包含"收入"的问题：
"生成低收入客群准入策略优化报告"
```

## 📋 报告内容结构

每份报告都包含以下完整结构：

### 1. 执行摘要
- 策略规则描述
- 数据规模和时间范围
- 综合评价（优秀/显著/良好）

### 2. 回测概况
- 回测时间段
- 样本总量
- 策略规则表达式

### 3. 核心指标对比
完整的新旧策略对比表格，包括：
- **通过率**：批准率变化
- **坏账率**：风险控制核心指标 🎯
- **KS值**：模型区分能力
- **通过人数**：实际通过数量
- **拒绝人数**：实际拒绝数量
- **坏账人数**：风险客户数量

### 4. 换手分析（Swap Set Analysis）
- 新策略额外拒绝的客户数
- 拦截准确率（实际坏账客户占比）
- 误伤率（误拒绝好客户比例）

### 5. 策略规则解释
- 代码表达式
- 白话翻译（业务友好）

### 6. 效果评估
- 综合评价等级（🌟🌟🌟 优秀 / 🌟🌟 显著 / 🌟 良好）
- 坏账率降低百分比
- KS值提升百分比
- 误伤率评估

### 7. 结论与建议
- 专业的策略上线建议
- 风险提示
- 监控建议

### 8. 附录：数据说明
- 数据来源说明
- 生成时间戳
- 计算标准说明

## 🚀 快速体验

### 方式1：使用快捷建议（推荐）

1. 打开 `workspace.html`
2. 点击顶部"报告生成"模块
3. 点击任一快捷建议按钮：
   - "生成年轻用户风险拦截策略回测报告"
   - "生成多头借贷客户拦截策略回测报告"
   - "生成低收入客群准入策略优化报告"

### 方式2：自然语言输入

直接在对话框中输入类似问题：
```
- "帮我生成一份年轻用户的风险策略报告"
- "分析多头借贷客户的策略效果"
- "生成收入相关的策略优化报告"
```

系统会智能识别关键词，自动选择对应的报告模板。

## ❓ 常见问题

### Q1: 报告数据是真实的吗？
**A**: 不是。目前使用的是精心设计的模拟数据，用于演示系统功能。实际生产环境需要对接真实的业务数据库。

### Q2: 如何生成不同类型的报告？
**A**: 在"报告生成"模块中，点击不同的快捷建议按钮，或在对话框中输入包含"年轻"、"多头"、"收入"等关键词的问题。

### Q3: 可以导出报告吗？
**A**: 导出按钮已预留，但功能尚未开发。未来将支持Word/PDF/PPT三种格式导出。

### Q4: 报告内容可以修改吗？
**A**: 目前报告是根据模板自动生成的。未来版本将支持在线编辑和自定义报告内容。

### Q5: 如何添加新的报告类型？
**A**: 需要在 `workspace.js` 的 `mockData` 中添加新的数据结构，然后在 `handleReportGeneration` 函数中添加相应的判断逻辑。
