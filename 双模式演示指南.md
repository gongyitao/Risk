# 双模式演示指南

## 🎯 3分钟快速演示两种模式

---

## 📺 演示1：自由模式（1分钟）

### 演示场景：快速生成报告

**演讲词**：
> "首先展示自由模式。假设我已经有了数据，只想快速生成一份策略报告。"

**操作步骤**：

1. **打开页面**
   ```
   打开 workspace.html
   默认就是自由模式
   ```

2. **直接跳到报告生成**
   ```
   点击顶部的"报告生成"按钮
   ```
   > "看，我可以直接跳到任何模块，不需要按顺序。"

3. **生成报告**
   ```
   点击"生成年轻用户风险拦截策略回测报告"
   ```
   > "点击一下，系统立即开始生成专业报告。"

4. **查看结果**
   ```
   右侧自动显示完整报告
   ```
   > "不到30秒，完整的策略分析报告就生成了。这就是自由模式的效率。"

**演示要点**：
- ✅ 强调灵活性和速度
- ✅ 展示可以跳过其他步骤
- ✅ 适合有明确目标的场景

---

## 📺 演示2：工作流模式（2分钟）

### 演示场景：完整策略分析流程

**演讲词**：
> "现在切换到工作流模式，展示如何完成一个完整的策略分析流程。"

**操作步骤**：

### 第1步：启用工作流模式（10秒）

1. **点击工作流按钮**
   ```
   点击顶部右侧的"工作流"按钮
   ```
   > "点击这里切换到工作流模式。"

2. **查看引导说明**
   ```
   左侧聊天区显示工作流说明
   ```
   > "系统会显示完整的工作流程：步骤1取数、步骤2分析、步骤3报告。"

3. **观察模块变化**
   ```
   顶部模块显示"步骤1"、"步骤2"、"步骤3"
   ```
   > "注意每个模块现在都标注了步骤编号。"

---

### 第2步：数据助手（30秒）

1. **输入需求**
   ```
   点击"查询近30天内逾期超过7天的用户..."
   ```
   > "步骤1：先取数。系统会自动生成SQL并执行。"

2. **查看进度**
   ```
   观察AI处理状态和右侧代码生成
   ```
   > "看，系统正在生成查询代码、执行、展示结果。"

3. **等待完成提示**
   ```
   2-3秒后显示"步骤完成"提示
   ```
   > "数据准备好了，系统自动询问是否进入下一步。"

---

### 第3步：策略挖掘（30秒）

1. **点击继续**
   ```
   点击"继续下一步"按钮
   ```
   > "点击继续，自动跳转到步骤2：策略挖掘。"

2. **输入分析需求**
   ```
   点击"帮我挖掘高风险客群..."
   ```
   > "现在进行特征分析和挖掘。"

3. **查看分析结果**
   ```
   右侧显示特征IV值图表
   ```
   > "系统自动计算156个特征，推荐TOP5高区分度特征。"

4. **等待完成提示**
   ```
   观察"步骤2"按钮出现✓标记
   ```
   > "步骤2完成，注意按钮上出现了✓标记。"

---

### 第4步：报告生成（30秒）

1. **点击继续**
   ```
   点击"继续下一步"按钮
   ```
   > "继续到步骤3：生成报告。"

2. **生成报告**
   ```
   点击快捷建议生成报告
   ```
   > "基于前面的数据和分析，生成专业报告。"

3. **查看完成**
   ```
   右侧显示完整报告
   ```
   > "完成！系统提示整个工作流已经完成。"

---

### 第5步：总结（10秒）

**演讲词**：
> "您看，工作流模式有三个特点：
> 1. **步骤清晰**：每一步都有标注和说明
> 2. **自动引导**：完成一步自动提示下一步
> 3. **进度可视**：完成的步骤有✓标记
> 
> 这对新手用户或需要完整流程的场景特别有用。"

## 🎬 完整演示脚本（5分钟版）

### 开场（30秒）

**演讲词**：
> "今天向大家展示策略E+系统的一个创新设计：双工作模式。

> - 老手希望够灵活

> - 有时只想快速查询
> 
> 所以我们设计了两种模式来满足这些需求。"

---

### 场景1：老手快速操作（1.5分钟）

**演讲词**：
> "假设我是一个熟练的分析师，现在需要快速查询数据。"

**演示步骤**：
1. 直接点击"数据助手"
2. 输入查询需求
3. 查看结果
4. **切换到"报告生成"**
5. 直接生成报告

**演讲词**：
> "看，我可以随意跳转，不需要按顺序。
> 这就是自由模式的优势：灵活、快速、高效。"

---

### 场景2：新手完整流程（2.5分钟）

**演讲词**：
> "现在假设我是一个新人，不太熟悉流程。我切换到工作流模式。"

**演示步骤**：
1. 点击"工作流"按钮
2. 阅读系统引导
3. **步骤1**：数据助手 - 取数
4. 点击"继续下一步"
5. **步骤2**：策略挖掘 - 分析
6. 点击"继续下一步"
7. **步骤3**：报告生成 - 出报告
8. 查看完成提示

**演讲词**：
> "工作流模式有三大优势：
> 1. 步骤清晰，不会迷路
> 2. 自动引导，不会遗漏
> 3. 进度可视，知道完成了多少
> 
> 这对新手培训和标准化流程特别有用。"

## ❓ 常见问题应对

### Q: "为什么要设计两种模式，不能自动判断吗？"

**A**: 
> "好问题！我们考虑过自动判断，但发现：
> 1. 用户偏好难以准确判断
> 2. 同一个用户在不同场景下需求不同
> 3. 给用户选择权更灵活
> 
> 所以我们采用了可切换的设计，一键就能切换。"

---

### Q: "工作流模式是强制的吗？可以跳步骤吗？"

**A**:
> "不是强制的！即使在工作流模式下，您也可以：
> 1. 随时点击任何模块
> 2. 跳过某些步骤
> 3. 回退到之前的步骤
> 
> 工作流只是提供引导，不是限制。"

---

### Q: "步骤完成的✓标记有什么用？"

**A**:
> "✓标记有两个作用：
> 1. **进度追踪**：一眼看出完成了哪些步骤
> 2. **质量保证**：确保不遗漏关键步骤
> 
> 这在培训新人和标准化流程时特别有用。"
